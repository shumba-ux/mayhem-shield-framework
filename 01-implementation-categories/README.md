# Implementation Categories

The framework uses one core methodology with six implementation categories.

Choose one primary category for each review, then add overlays for material deployment complexity.

## Categories
1. Pure AI Services
2. AI-Native SaaS
3. Traditional SaaS with AI Features
4. SaaS with AI Enhancement
5. Infrastructure with AI
6. AI-Native Content Generation

## EU AI Act risk-tier mapping

Each implementation category typically aligns with one or more EU AI Act risk tiers. Final risk-tier classification is use-case dependent and is confirmed during scoping based on the deployment's purpose, user base, and regulatory context. The table below describes typical alignment, not a deterministic rule.

| Implementation Category | Typical EU AI Act risk tier |
|---|---|
| Pure AI Services | High-risk or Limited-risk depending on use case (e.g., HR decisioning vs. content generation) |
| AI-Native SaaS | High-risk in regulated contexts; Limited-risk for general productivity use |
| Traditional SaaS with AI Features | Limited-risk or Minimal-risk |
| SaaS with AI Enhancement | Minimal-risk |
| Infrastructure with AI | Variable; depends on the application built on top |
| AI-Native Content Generation | Limited-risk by default (transparency obligations); High-risk for credit, hiring, or other regulated decisioning use |

Where the EU AI Act applies, the risk tier is documented as part of intake and informs review depth, control findings, and gate conditions. See `../00-core-framework/framework-alignment.md` for broader EU AI Act alignment context.
